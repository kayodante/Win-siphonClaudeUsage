# Support

## Bug or feature request

Open a [GitHub Issue](../../issues/new/choose) and pick the matching
template. Bug reports go faster to a fix when they include the Siphon
version, Windows version, and how you installed it (installer, portable, or
winget).

## Usage question

Not a bug — just wondering how something works or how to install it? Check
the [README](README.md) first: it covers installation (installer, portable,
winget), features, and how sign-in works. If it's a winget-specific install
issue, note that new releases can lag GitHub Releases by a few days while
[microsoft/winget-pkgs](https://github.com/microsoft/winget-pkgs) approval
goes through.

If the README doesn't answer it, open a
[GitHub Issue](../../issues/new/choose) anyway and describe your question —
just don't file it as a bug report if nothing is actually broken.

## Security issue

Do not use Issues for this — see [SECURITY.md](SECURITY.md).
