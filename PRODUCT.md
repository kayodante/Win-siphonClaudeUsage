# Product

## Register

product

## Users

Claude Code developers on Windows who need to monitor their 5-hour session quota, weekly limit, and token costs without breaking flow. The app lives in the system tray — they glance at it mid-session, not sit in front of it. Context: second monitor, taskbar, background awareness. Primary task on any given screen: knowing how much quota remains and when it resets.

## Product Purpose

Real-time Claude Code usage monitor. Surfaces session %, weekly %, estimated costs, and arms reset notifications. Ambient by design — always running, rarely in focus, never in the way. Success means a developer never gets surprised by a quota cutoff mid-task.

## Brand Personality

Sharp, electric, alive. Data feels live. Numbers have urgency without anxiety. The app knows things before you think to ask.

## Anti-references

- Purple/blue glow aesthetics, glassmorphism, neon accents — dark mode as an aesthetic statement rather than function
- Generic SaaS dashboard patterns: pastel cards, hero metrics with gradient text, identical card grids

## Design Principles

1. **Data over decoration** — every visual element earns its place by communicating state, not dressing the screen
2. **Ambient precision** — the app feels like it's always watching, always accurate; it rewards a glance, not a read
3. **State is visible** — quota level, reset timing, online/offline status should be immediately readable without scanning
4. **Urgency without alarm** — high usage reads as urgent, not stressful; the user should feel informed, not panicked
5. **Native, not ported** — feels built for Windows power users; the pixel font and live data aesthetic are deliberate, not coincidental

## Accessibility & Inclusion

WCAG AA minimum. Color is already used semantically (level-ok/warn/high/critical) — motion and shape must reinforce state, not replace color for users who cannot perceive it. Respect `prefers-reduced-motion`.
